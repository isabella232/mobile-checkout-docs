  • MappedNetworkInput

    Allows the use of mapped data of any specified key value types for use as input to NetworkOperations. The way this is specified is through calls to SDK.performIO. There are some new overloads that allow users to specify these values.

    Example

    SDK.performIO('actionName', input: MappedNetworkInput<String,String>(input: [
 "key": "value"
]) { ... }

    Additionally a convenience method has been created for use with the SDK class and mapped input data when only a single pair is in use.

    SDK.performIO('actionName', key: "id", value: 234) { ... }

// which is the same thing as

SDK.performIO('actionName', input: MappedNetworkInput<String,Int>([
 "id": 234
])) { ... }
    See more

    Declaration

    Swift

    public class MappedNetworkInput<Key: Hashable, Value>:
NetworkInput, ExpressibleByDictionaryLiteral
  • OrderedNetworkInput

    Allows the use of ordered data of any specified type for use as input to NetworkOperations. The way this is specified is through calls to SDK.performIO. There are some new overloads that allow users to specify these values.

    Example

    SDK.performIO('actionName', input: OrderedNetworkInput<String>(
  "string1",
  "string2",
  "string3"
)) { ... }

    Additionally a convenience method has been created for use with the SDK class and ordered input sets. Instead, simply call:

    SDK.performIO('actionName', values: [3, 4, 5]) { ... }

// which is the same thing as

SDK.performIO('actionName', input: OrderedNetworkInput<Int>(3, 4, 5)) { ... }
    See more

  • SequenceProcessor

    The SequenceProcessor is designed to take a list of callback instances and execute the first one while providing it the ability to execute the next item in the list. This style of sequencing allows for truly asynchronous execution with each closure either passing the Next instance off to another function or class to be executed later or to be ignored if logic in that step indicates that progress should not continue.

    Each closure must execute the Next closure for the chain to continue. Passing an error to the closure will cause execution to halt.
