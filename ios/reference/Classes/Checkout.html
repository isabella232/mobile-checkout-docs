  • performIO(actionName:input:queue:completion:)

    NetworkOperations registered with an actionName can be called from any place within the execution of the SDK. This is ideally used to allow 1st party host apps expose custom functionality that will likely be used from within their custom UI components.

    If a NetworkOperation is registered as an ActionBlock with the actionName of ‘FetchFIs’ then it can be later referenced and invoked via

    extendedConfig.networkOperations.append(
  NetworkOperation("name", type: .ActionBlock, actionName: "doIt") {
    callback in

    callback?(NetworkOperationResult(.Success))
  }
)

SDK.performIO('doIt') { networkResult in
  // evaluate the result of the call
  switch networkResult.status {
    case .Success:
      // ...
  }
}

    Note

    Synchronicity is ignored for all ActionBlock style NetworkOperations as they are executed asynchronously by default on the main thread.

  • performIO(actionName:map:queue:completion:)

    A convenience function with a different signature for performIO. See the above base function for more information if necessary. This function takes a single key/value pair and constructs an instance of MappedNetworkInput to contain it. This is then passed to the base performIO directly.

    Note

    In cases where multiple keys and values are desired, please use performIO(actionName, input: MappedNetworkInput<K,V>([:])) instead

  • performIO(actionName:list:queue:completion:)

    A convenience function with a different signature for performIO. See the above base function for more information if necessary. This function takes an array of values and constructs an instance of OrderedNetworkInput to contain them. This is then passed to the base performIO directly.
