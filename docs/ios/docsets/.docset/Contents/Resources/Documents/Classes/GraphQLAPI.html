  • authedFetch(authToken:clientId:query:serviceKey:cachePolicy:queue:resultHandler:)

    Fetches a query from the server or from the local cache, depending on the current contents of the cache and the specified cache policy. This is a delegated method, making it similar to calling GraphQLAPI.shared.client.fetch(...).

    The difference between this function and fetch() is that it takes an authToken and clientId which are used to generate a copy of the specified GraphQLService instance with the specifeid headers already set.

    Note

    The resultHandler has the following signature ( _ result: GraphQLResult<Operation.Data>?, _ error: Error? ) -> Void

    Declaration

    Swift

    @discardableResult
public static func authedFetch<Query: GraphQLQuery>(
  authToken: String,
  clientId: String,
  query: Query,
  serviceKey: PayPalEnvironment = .live,
  cachePolicy: CachePolicy = .returnCacheDataElseFetch,
  queue: DispatchQueue = DispatchQueue.global(qos: .userInitiated),
  resultHandler: GraphQLResultHandler<Query>? = nil
) throws -> Cancellable

    Parameters

    authToken

    An authorization token to send to the remote GraphQL service
    clientId

    A client id string representing the merchant making the call
    query

    The query to fetch.
    cachePolicy

    A cache policy that specifies when results should be fetched from the server and when data should be loaded from the local cache.
    queue

    A dispatch queue on which the result handler will be called. Defaults to the main queue.
    resultHandler

    An optional closure that is called when query results are available or when an error occurs.

    Return Value

    An object that can be used to cancel an in progress fetch.
