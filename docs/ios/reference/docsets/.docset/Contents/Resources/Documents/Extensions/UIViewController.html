  • isInContainerView()

    Checks to see if the UIViewController’s parent is a container view or not. This is done by the process of elimination. If there is a parent but that parent is none of navigationController, splitViewController, tabBarController, or popoverPresentationController then true is returned.

    Declaration

    Swift

    public func isInContainerView() -> Bool

    Return Value

    true if in a container view, false otherwise

  • adjustContainerViewFramesIfAny(updateHeight:updateWidth:updateX:updateY:usingHeight:)

    When a UIViewController is being presented from within a container view (usually setup via Interface Builder), this will place a KVO observer on the bounds of the view controller’s view and update the superview’s bounds accordingly.

    By default, both the width and height are adjusted to match that of the view controller’s view, but this can be controlled by passing false to either updateWidth or updateHeight.

    Declaration

    Swift

    public func adjustContainerViewFramesIfAny(
  updateHeight: Bool = true,
  updateWidth: Bool = false,
  updateX: Bool = false,
  updateY: Bool = false,
  usingHeight: CGFloat? = nil
)

    Parameters

    updateWidth

    true if the width of the containing view should be modified.
    updateHeight

    true if the height of the containing view should be modified.
